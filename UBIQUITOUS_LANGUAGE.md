# Ubiquitous Language — ext-shifting

Canonical terminology for this codebase and any tooling built around it.

---

## Terms

| Term | Definition | Aliases to avoid |
|------|------------|-----------------|
| **Simplicial complex** | A collection of simplices closed under taking subsets. In this codebase, represented as a list of lists of non-negative integers (vertex labels). Variable name `cplx` in code. | "complex" (ambiguous with algebraic complex), `complex` (code variable — prefer `cplx`) |
| **Simplex** | A single face of a simplicial complex — a sorted list of vertex labels, e.g. `{1,2,3}`. The *dimension* of a simplex is its length minus one. | "face", "element" |
| **Vertex** | A non-negative integer label representing a point in a simplicial complex. | "node", "point" |
| **k-skeleton** | The sub-collection of all simplices of dimension ≤ k. E.g. the 1-skeleton contains all edges and vertices. | "skeleton of dimension k" |
| **Surface** | A 2-dimensional closed simplicial complex without boundary — a triangulation of a topological surface. All simplices are triangles. | "2-complex", "mesh" |
| **Triangulation** | A specific surface represented as a list of triangles (3-element simplices). Distinct from the abstract topological surface it represents. In the analysis workflow, `triangulations` is a list of these. | "surface" (when referring specifically to the combinatorial data) |
| **Exterior shifting** | An operation that maps a simplicial complex to a shifted complex using compound matrices derived from a random matrix. The result depends on a chosen vertex ordering. The canonical function is `exteriorShift`. | "shifting" (acceptable shorthand), "ext shift" |
| **Shifted complex** | The output of exterior shifting. A simplicial complex that is *shifted*: if a simplex is in the complex, any simplex obtained by replacing a vertex with a smaller-indexed vertex is also in the complex. | "shift" (as noun — acceptable shorthand) |
| **Lex order** | A total ordering of k-element subsets of vertices by lexicographic ordering. Used as the `fullOrder` argument to `exteriorShift`. Generated by `LexOrder(v, k)`. | "lexicographic order", "lex" (acceptable abbreviation) |
| **Rev-lex order** | A total ordering of k-element subsets of vertices by reverse lexicographic ordering. Generated by `RevLexOrder(v, k)`. | "reverse lex", "revlex" (acceptable abbreviation) |
| **Full order** | A complete ordering of all k-element subsets of vertices, used as the reference ordering for exterior shifting. Must contain all vertices appearing in the input simplices. | "vertex ordering", "ordering" |
| **Compound matrix** | A matrix whose columns are indexed by k-element subsets, with entries given by the corresponding k×k minor of the input matrix. Core computational object in exterior shifting. | "minor matrix" |
| **Vertex split** | An operation on a surface that replaces one vertex with two new vertices connected by an edge, increasing the vertex count by one while preserving the topological type. The *base* vertex is the one being split. | "split" (ambiguous — see triangle split) |
| **Triangle split** | An operation that replaces one triangle with three triangles by inserting a new vertex at its centre. Distinct from vertex split. | "split" (ambiguous — see vertex split) |
| **Split base** | The vertex being split in a vertex split. Stored as `SPLITBASE = 0` index in split info. | "source vertex", "origin vertex" |
| **Split neighbors** | The two vertices that become the endpoints of the new edge introduced by a vertex split. Stored at `SPLITNEIGHBORS = 1` index in split info. | "endpoints" |
| **Triangle ratio** | The pair `{side1, side2}` describing how the triangles around the split base are distributed between the two new vertices after a vertex split. Stored at `SPLITRATIO = 2` index in split info. | "split ratio" |
| **Non-trivial split** | A vertex split that is not a trivial (minimal) split — specifically, one where the triangle ratio is not `{1, n-1}`. These are the splits passed to the next analysis iteration. | "interesting split", "nontrivial split" |
| **Critical region** | A structural pattern in the shifting of a triangulation that signals non-standard shifting behaviour. Identified by type (e.g. "disk"), a vertex count, and an index. | "critical area", "crit region" |
| **Final edge of shift** | The last edge (2-simplex of dimension 1) in the exterior shifted complex under the chosen ordering. Used as a diagnostic to identify critical regions. Computed by `finalEdgeOfShift`. | "last edge", "terminal edge" |
| **Prefix** | A shifted complex is said to be a *prefix* if it consists of the first N simplices in the full order. A triangulation whose shifting is not a prefix is considered structurally interesting. | "initial segment" |
| **Analysis run** | A named batch computation: starts from an input set of triangulations, runs exterior shifting and critical region analysis, and produces output + a new input set for the next iteration. Named via `analysisName`. | "experiment", "job" |
| **Iteration** | One execution of `analyze triangs.m2`. Processes the current input file, writes results, and produces the input file for the next iteration. Convergence occurs when no further splits are generated. | "pass", "step" |
| **Contractible edge** | An edge in a surface whose contraction (identifying its two endpoints) yields a topologically equivalent surface. | "collapsible edge" |
| **Euler characteristic** | A topological invariant of a surface, computed as: #triangles + #vertices − #edges. Computed by `eulerCharSrfc`. | "Euler number" |
| **Torus** | The topological surface of genus 1. The standard starting triangulation is in `tori.m2` / `irred tori.m2`. | "T", "T²" |
| **Klein bottle** | A non-orientable closed surface. Starting triangulations in `klein bottle.m2` / `irred kb.m2`. | "KB" (acceptable abbreviation) |
| **Projective plane** | The non-orientable closed surface of Euler characteristic 1. Triangulations in `proj plane.m2` / `irred pp.m2`. | "RP²", "PP" |

---

## Relationships

- A **triangulation** is a **simplicial complex** where every maximal simplex is a **triangle** (2-simplex).
- A **surface** is a **triangulation** that is closed (no boundary edges) and connected.
- **Exterior shifting** takes a set of simplices of the same dimension and a **full order**, and returns a **shifted complex** of the same dimension using a **compound matrix**.
- A **vertex split** on a **triangulation** produces a new **triangulation** with one more vertex. The metadata (split info) records the **split base**, **split neighbors**, and **triangle ratio**.
- An **analysis run** consists of multiple **iterations**. Each iteration reads a list of **triangulations**, computes the **final edge of shift** for each, identifies **critical regions**, and generates **non-trivial splits** as input for the next iteration.
- **Critical regions** are identified from the relationship between a triangulation and its **final edge of shift**.

## Example Dialogue

> "I ran `extShiftLex` on the edges of the Klein bottle triangulation and the **final edge of shift** didn't land in the expected position, which suggests there's a **critical region** involving a disk of 3 vertices."

> "The **analysis run** for the torus completed in 3 **iterations** — the first iteration found some **non-trivial splits**, the second processed those, and the third produced no further splits, so the calculation converged."

> "We need to do a **vertex split** on the **split base** vertex 4, with **split neighbors** {2, 6} and a **triangle ratio** of {3, 4}."

## Flagged Ambiguities

| Term | Ambiguity | Recommended resolution |
|------|-----------|----------------------|
| `split` | Used in code for both **vertex split** and **triangle split**, which are distinct operations | Always qualify: "vertex split" or "triangle split" |
| `complex` / `cplx` | Used as a variable name for any simplicial complex, but "complex" alone is ambiguous (could mean algebraic chain complex) | Use "simplicial complex" in prose; `cplx` in code |
| `surface` | Sometimes means the abstract topological surface (torus, Klein bottle), sometimes the specific triangulation data | Use "surface" for the topological object, "triangulation" for the combinatorial data |
| `shift` (noun) | Could refer to the operation or the result | Use "exterior shifting" for the operation, "shifted complex" for the result |
| `iteration` | Could mean one call to `analyze triangs.m2`, or one step within the shifting algorithm | Use "analysis iteration" for the former, "shifting step" for the latter if needed |
| `dimension` | Could refer to the dimension of an individual simplex (length − 1) or the dimension of the whole complex | Qualify as "simplex dimension" or "complex dimension" when ambiguous |
